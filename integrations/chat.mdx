Chat platforms are linked from the **Request\+ Web Dashboard**, not inside the desktop app. The dashboard handles OAuth, stores the connection on your account, and the desktop client picks it up automatically.

<Card title="Open the Request+ Dashboard" icon="arrow-up-right-from-square" href="https://requestplus.xyz/dashboard">
  Sign in to manage your platform connections.
</Card>

## Platform Connections

In the dashboard you'll find the **Platform Connections** panel with a card for each supported service:

<Frame>
  ![Request\+ dashboard showing Platform Connections panel with Twitch, Kick, YouTube, and Patreon cards.](/images/dashboard-platform-connections.png)
</Frame>

| Platform | Use | Status |
| --- | --- | --- |
| Twitch | Listen to Twitch chat for `!sr` | Stable |
| Kick | Listen to Kick chat for `!sr` | Stable |
| YouTube | Listen to YouTube live chat | **Experimental** — the YouTube Data API is paid, so quota limits apply and reliability varies |
| Patreon | Optional perks / supporter sync | Optional |

<Warning>
  **YouTube is experimental.** Because the YouTube Data API is a paid Google service, chat polling is rate-limited and may stop responding during long streams or when daily quota is exhausted. Use Twitch or Kick as your primary chat where possible.
</Warning>

## Connect a platform

<Steps>
  <Step title="Open the dashboard">
    Go to [requestplus.xyz/dashboard](https://requestplus.xyz/dashboard) and sign in.
  </Step>
  <Step title="Click Connect on the platform card">
    In the **Platform Connections** panel, click **Connect** on Twitch, Kick, or YouTube.
  </Step>
  <Step title="Follow the on-screen OAuth instructions">
    A new tab opens on the platform's official site (Twitch, Kick, or Google for YouTube). Follow the on-screen prompts:

    - Sign in if you aren't already.
    - Review the requested scopes — Request\+ asks for **read access to chat** and the minimum needed to identify your channel.
    - Click **Authorize** / **Allow**.

    You'll be redirected back to the Request\+ dashboard automatically.
  </Step>
  <Step title="Confirm the green check">
    On the platform card, you should now see **Connected and active** with a green check. For YouTube, your channel name (e.g., `DayTrack`) appears below the status.
  </Step>
  <Step title="Restart the desktop client (important)">
    The desktop client only picks up new chat connections at startup.

    <Warning>
      If you connect or reconnect a chat platform while Request\+ is already running, **chat will not be listened to** until you fully restart the app — and that means closing it from the system tray, not just the window. See [Closing Request\+](/desktop/closing-the-app).
    </Warning>
  </Step>
</Steps>

## Disconnect a platform

Click **Disconnect** on the platform's card in the dashboard. The OAuth token is revoked on Request\+'s side immediately. Restart the desktop client so it stops trying to listen to that chat.

## Chat commands

Viewers request songs with `!sr` (song request). Request\+ accepts plain text searches or direct platform URLs:

```text
!sr never gonna give you up
!sr https://open.spotify.com/track/...
!sr https://music.youtube.com/watch?v=...
!sr https://music.apple.com/...
!sr https://soundcloud.com/...
```

The desktop client resolves the request on your active music platform, updates the queue, and reports status back to chat (if responses are enabled).

## Moderation rules

From **Settings → Requests** in the desktop app, you can restrict who can request:

- **Moderators only** — only chat mods can request.
- **Subscribers only** — only subs can request (Twitch/YouTube).
- **Manual approval** — every request waits in a pending queue until you approve.
- **Guess the Song** — hide track info from the overlay until the song is revealed.

See [Queue & moderation](/queue) for the full breakdown.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Chat isn't being read after I connected">
    The desktop client caches platform connections on startup. **Quit Request\+ from the tray** and reopen it. See [Closing Request\+](/desktop/closing-the-app).
  </Accordion>

  <Accordion title="YouTube chat stopped mid-stream">
    YouTube's API has daily quota. When quota is exhausted, polling stops until it resets (usually 24 h). This is a YouTube limitation, not a Request\+ bug.
  </Accordion>

  <Accordion title="Dashboard shows connected but no requests come through">
    Confirm the desktop client is running and signed in to the same Request\+ account, then restart it from the tray.
  </Accordion>
</AccordionGroup>
